# Content Review Workflow

1. Author creates draft content with skill mapping, objectives, examples, checks, safety guardrails, and teacher notes.
2. Automated validation checks completeness, unsafe wording, empty mappings, and missing explanations.
3. Reviewer checks learner usefulness, safety framing, age appropriateness, and classroom clarity.
4. Content moves to `reviewed` only with reviewer/date/notes.
5. Product owner publishes only when content has linked practice, review cadence, and no safety issues.
6. Feedback and analytics can mark content `needs_update` when students repeatedly miss the same concept or report confusion.
7. Revisions must update version history and preserve previous reviewer notes.

## Admin dashboard requirements

The admin dashboard should prioritize: lessons needing review, confusing lessons, low-usefulness lessons, labs with low scores, exercises with high wrong-answer rate, weak skills by cohort, and content gaps from tutor/support questions.
