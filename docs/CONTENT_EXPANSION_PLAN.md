# Content Expansion Plan

CyberPath is not Khan Academy-level yet. This plan defines how it can grow toward that quality without filler.

## Beta content target

- 50 strong lessons
- 300 exercises
- 20 safe labs
- 10 guided projects
- 120 glossary terms
- 9 skill-tree categories
- 50+ skill nodes

## Lesson quality standard

Each published lesson should include title, skillId, prerequisites, objectives, beginner explanation, analogy, example, common mistake, mini-check, quiz/exercises, related lab, portfolio artifact suggestion, glossary terms, and review schedule.

## Lab quality standard

Each lab must include scenario briefing, fictional dataset, safe guardrails, evidence checklist, tasks, hints, rubric, feedback, solution outline, and artifact suggestion.

## Guided project standard

Each project should include goal, prerequisites, steps, checkpoints, rubric, artifact output, and mentor review criteria.

## Admin content lifecycle

Content should support draft, reviewed, published, and needs_update statuses; review date; content quality score; feedback summary; publish/unpublish; and version history.

## Unsafe content filter

Published content must be checked for unsafe patterns related to credential theft, malware, evasion, persistence, phishing creation, live-target exploitation, bypassing, data theft, or unauthorized access.
