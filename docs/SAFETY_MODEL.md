# CyberPath Academy Safety Model

CyberPath Academy is defensive-only. It must not teach credential theft, malware, persistence, evasion, live-target exploitation, phishing creation, bypassing security systems, stealing data, or unauthorized access.

## Allowed learning activities
- Defensive analysis of fictional data.
- Evidence collection and incident documentation.
- Detection, prevention, hardening, and remediation reasoning.
- Access reviews, password policy audits, toy secure-code review, and incident report writing.
- High-level conceptual explanations that redirect unsafe requests toward defender workflows.

## Lab boundaries
All labs must use fictional datasets, toy examples, explicit guardrails, and authorization reminders. Learners should identify facts, explain risk, choose safe next steps, communicate clearly, and avoid harmful assumptions.

## Tutor boundaries
The guided tutor fallback refuses unsafe offensive requests and redirects to defensive learning. Unsafe requests and content gaps are logged for admin review so the curriculum can improve without expanding into harmful workflows.

## Public portfolio boundaries
Published artifacts must be read-only, fictional, and free of private user data. Artifacts should demonstrate professional defensive thinking, not offensive procedure.
