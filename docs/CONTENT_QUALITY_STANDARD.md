# Content Quality Standard

## Publishing bar

Published lessons must include:

- `skillId` mapping through the skill tree or track links.
- prerequisites or explicit beginner entry status.
- 3+ learning objectives.
- beginner explanation using safe defensive language.
- analogy or concrete beginner framing.
- worked example using fictional/toy data only.
- common mistake and correction.
- at least one mini-check or knowledge check.
- related exercises.
- related lab or clear reason no lab is needed yet.
- related portfolio artifact suggestion.
- glossary terms.
- review cadence.
- teacher note.

## Status model

Content status values:

- `draft`: authoring in progress; not student-visible unless explicitly demo-only.
- `reviewed`: checked by a reviewer but not yet final.
- `published`: visible to learners and included in mastery/recommendations.
- `needs_update`: visible only if still accurate and flagged for revision.

## Review metadata

Each content object should track reviewer, last reviewed date, quality score, learner usefulness score, confusion notes, and revision history. Scores must be treated as editorial signals, not proof of learning until real learner data validates them.

## Safety validation

Reject or revise content that includes credential theft, malware, evasion, persistence, real-target exploitation, bypass instructions, stealing data, phishing creation, or unauthorized access. Use fictional datasets, defensive analysis, reporting, prevention, and authorized incident-response framing.
