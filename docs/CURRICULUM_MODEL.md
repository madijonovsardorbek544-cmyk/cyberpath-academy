# CyberPath Academy Curriculum Model

CyberPath content is structured around role readiness, safe practice, and portfolio evidence.

## Target scale
- 100 lessons
- 500 quiz questions
- 25 safe labs
- 10 guided projects
- 150 glossary terms
- Role paths for Cyber Foundations, Networking, Linux/Terminal, Blue Team/SOC Level 1, Web Security Defense, Cloud Security Basics, Security Engineering Basics, GRC/Risk Basics, and AI Security Awareness.

## Lesson structure
Each lesson should include prerequisites, learning objectives, plain explanation, analogies/visuals, key terms, examples, knowledge checks, quiz questions, common mistakes, defensive real-world connection, portfolio relevance, review status, and version metadata.

## Lab structure
Each lab should include scenario briefing, fictional dataset, safe guardrails, tasks, expected evidence, rubric criteria, hints, solution outline, retry path, and submission feedback.

Rubric categories:
1. Evidence identification
2. Risk reasoning
3. Defensive next step
4. Clarity
5. Safety / authorization

## Artifact structure
Portfolio artifacts should include title, artifact type, summary, scenario, evidence used, risk explanation, defensive recommendations, reflection, created date, and optional linked lab/project.

## Adding content safely
New content should be reviewed against `docs/SAFETY_MODEL.md`. If an example looks like it could enable unauthorized activity, rewrite it into detection, prevention, documentation, or incident-response framing.
