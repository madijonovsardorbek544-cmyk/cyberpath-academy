# Pilot Success Metrics

CyberPath Academy beta success is measured by behavior and willingness-to-pay signals, not vanity traffic.

## Core metrics

| Metric | Target for 30-user beta | Why it matters |
| --- | ---: | --- |
| Activation rate | 20/30 onboarding completions | Students understand the start. |
| Lesson completion rate | 15/30 complete one lesson | First learning value is reachable. |
| Practice completion rate | 15/30 complete one practice/review session | Learners can apply concepts. |
| Lab submission rate | 10/30 submit a safe lab | Labs are usable without founder help. |
| Artifact creation rate | 5/30 create artifacts | Proof-of-work loop is credible. |
| Feedback rate | 20 feedback submissions | Product can learn from users. |
| Teacher usefulness score | Average 4/5 from teachers/mentors | Dashboard is understandable. |
| Willingness-to-pay signal | 1 serious pilot lead or 3 maybe/yes signals | Pilot can become revenue. |

## Operational metrics

- Bugs reported by severity.
- Users who get stuck before onboarding completion.
- Most abandoned page.
- Most confusing lesson or lab.
- Top requested features.

## Go/no-go interpretation

- **Continue beta:** users complete the core loop but report fixable confusion.
- **Pause and fix:** users cannot finish onboarding, lesson, lab, or artifact without help.
- **Do not sell publicly yet:** unresolved safety, privacy, billing, or school administration gaps remain.
