# Manual Browser QA Script for Controlled Beta

_Last updated: 2026-05-17._

Use this because Playwright browser E2E is blocked by registry policy. Run with the public mock demo or local mock mode. Record device, browser, viewport, tester role, and whether the tester needed founder help.

## Student 30-minute script

1. Open public demo.
2. Create account.
3. Finish onboarding.
4. Open dashboard.
5. Open skill tree.
6. Complete one practice question.
7. Open review.
8. Complete one lesson.
9. Open one lab.
10. Submit lab.
11. Create artifact.
12. Submit feedback.
13. Report one confusing thing.

## Teacher 20-minute script

1. Open mentor demo.
2. Inspect cohort dashboard.
3. Find weakest students.
4. Inspect heatmap.
5. Review lab submissions.
6. Review artifacts.
7. Export CSV.
8. Submit school pilot interest.
9. Answer whether they would use this in class.

## Admin 15-minute script

1. Open admin demo.
2. Inspect beta readiness.
3. Inspect bug reports.
4. Inspect feedback.
5. Inspect pilot leads.
6. Inspect validation metrics.

## End questions

- Would pay? yes / maybe / no
- Would recommend to a classmate or teacher? yes / maybe / no
- What was the first confusing moment?
- What action did you expect but could not find?
- Did anything feel unsafe or too advanced?
- Did the page explain itself without the founder?
